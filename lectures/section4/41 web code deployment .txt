المحاضرة 41 - رفع المواقع على الإنترنت (Deployment)
ملف: lec 41 deployment notes

------------------------------------------------------------
موضوع المحاضرة 41
------------------------------------------------------------
كيفية رفع موقع ويب على الإنترنت مجاناً
الملف المستخدم في الشرح: موقع myTunes


------------------------------------------------------------
أولاً: لماذا نحتاج إلى رفع الموقع؟
------------------------------------------------------------

عندما تفتح ملف index.html من جهازك مباشرة:
  - العنوان يكون:   file:///C:/مجلدك/index.html
  - لا يستطيع أحد رؤيته غيرك
  - غير موجود على الإنترنت

عندما ترفع الموقع على خدمة استضافة:
  - العنوان يصبح:   https://اسمك.netlify.app
  - أي شخص في العالم يستطيع فتحه
  - موجود على الإنترنت 24 ساعة


------------------------------------------------------------
ثانياً: نوعا المواقع من حيث الحاجة للاستضافة
------------------------------------------------------------

مواقع كبيرة (تحتاج استضافة مدفوعة):
  - مواقع بها قواعد بيانات (database)
  - مواقع بها لغة خادم (PHP, Node.js, Python)
  - مثال: متجر إلكتروني، شبكة اجتماعية

مواقع صغيرة / ثابتة (static) - مجانية:
  - تتكون فقط من HTML + CSS + JavaScript
  - مثل موقع myTunes
  - يمكن رفعها مجاناً على Netlify أو GitHub Pages


------------------------------------------------------------
ثالثاً: الخدمتان المجانيتان اللتان ذكرهما الأستاذ أحمد
------------------------------------------------------------

الخدمة الأولى: Netlify
  الرابط:    https://www.netlify.com/
  المميزات:
    - الأسهل للمبتدئين
    - رفع الملفات بالسحب والإفلات (drag and drop)
    - يمنحك رابطاً فورياً مثل: اسمعشوائي.netlify.app
    - خدمات كثيرة: نماذج، وظائف، نطاقات مخصصة
    - الخطة المجانية كافية لمشاريع التعلم
  قال الأستاذ: اختار هذه لأنها الأسهل

الخدمة الثانية: GitHub Pages
  الرابط:    https://pages.github.com/
  المميزات:
    - تابعة لـ GitHub التي تملكها Microsoft
    - قال الأستاذ: Microsoft محترفون في البرمجة
    - تتكامل مع Git مباشرة
    - مناسبة أكثر لمن يعرف Git بشكل جيد
  قال الأستاذ: لم يشرحها لكنه لم يقلل منها


------------------------------------------------------------
رابعاً: خطوات رفع الموقع على Netlify (كما شرحها الأستاذ)
------------------------------------------------------------

الخطوة 1 - إنشاء حساب على Netlify
  1. افتح:      https://www.netlify.com/
  2. اضغط:      Sign up
  3. اختر طريقة التسجيل (ظهرت هذه الخيارات في الشرح):
       - GitHub   (الأفضل إذا كان لديك حساب GitHub)
       - GitLab
       - Bitbucket
       - Email    (بالبريد الإلكتروني العادي)

الخطوة 2 - الدخول إلى لوحة التحكم
  بعد تسجيل الدخول تظهر صفحة:
    "ahmed samy alkhouly's team > Sites"
  وتجد زر: New site from Git

الخطوة 3 - رفع الملفات بالسحب والإفلات
  في أسفل الصفحة يوجد مربع نصه:
    "Want to deploy a new site without connecting to Git?
     Drag and drop your site folder here"
  قام الأستاذ بسحب مجلد الموقع كاملاً وإفلاته هناك
  ظهرت رسالة: Uploading...

الخطوة 4 - انتظار الرفع
  بعد الرفع تنتقل تلقائياً لصفحة الموقع الجديد
  يعطيك Netlify اسماً عشوائياً مثل:
    vigilant-thompson-f51ca9
  والرابط يصبح:
    https://vigilant-thompson-f51ca9.netlify.app

الخطوة 5 - فتح الموقع
  اضغط على الرابط لفتح موقعك المنشور على الإنترنت
  أي شخص الآن يستطيع فتح هذا الرابط


------------------------------------------------------------
خامساً: ما رفعه محمود بنجاح
------------------------------------------------------------

الموقع المرفوع: myTunes (موقع المحاضرة 40-41)
الملفات المرفوعة:
  - index.html
  - main.css

الرابط بعد الرفع:
  https://vigilant-thompson-f51ca9.netlify.app

ملاحظة: هذا رابط تجريبي من إنشاء Netlify تلقائياً.
يمكن تغييره لاحقاً من إعدادات الموقع (Site settings).


------------------------------------------------------------
سادساً: الفرق بين الرفع بـ Git والرفع اليدوي
------------------------------------------------------------

الرفع اليدوي (ما شرحه الأستاذ):
  - تسحب المجلد وتفلته في Netlify
  - سريع ومباشر
  - لا يتحدث تلقائياً عند تعديل الكود
  - تحتاج لرفع يدوي جديد عند كل تعديل

الرفع بـ Git (New site from Git):
  - يربط Netlify بـ GitHub repository
  - عند كل push للكود → يتحدث الموقع تلقائياً
  - الطريقة الاحترافية في العمل الفعلي
  - تحتاج معرفة Git


------------------------------------------------------------
سابعاً: ملاحظات مهمة
------------------------------------------------------------

1. لا تضع الملفات داخل مجلد فرعي:
   index.html يجب أن يكون في جذر المجلد الذي ترفعه
   مثال صحيح:   مجلدي/index.html
   مثال خاطئ:   مجلدي/موقعي/index.html

2. تأكد من رفع كل الملفات معاً:
   index.html + main.css في نفس المجلد

3. الرابط الأول عشوائي:
   يمكن تغييره لاحقاً من: Site settings > Domain settings

4. الخطة المجانية كافية تماماً لمشاريع التعلم


------------------------------------------------------------
ثامناً: مقارنة سريعة
------------------------------------------------------------

                    Netlify           GitHub Pages
السهولة:            الأسهل            تحتاج Git
الرفع:              سحب وإفلات        عبر GitHub
التحديث التلقائي:   مع Git            دائماً
الجودة:             ممتازة            ممتازة (Microsoft)
المجانية:           نعم               نعم


------------------------------------------------------------
تاسعاً: ملخص المحاضرة
------------------------------------------------------------

تعلمنا في هذه المحاضرة:
  - المواقع الثابتة (HTML + CSS) يمكن رفعها مجاناً
  - Netlify هي الأسهل للمبتدئين
  - الرفع اليدوي: سحب المجلد وإفلاته في Netlify
  - بعد الرفع تحصل فوراً على رابط إنترنت حقيقي
  - GitHub Pages بديل قوي خاصة مع معرفة Git


------------------------------------------------------------
سؤال المراجعة
------------------------------------------------------------

ما الفرق الرئيسي بين الرفع اليدوي والرفع عبر Git في Netlify؟

الجواب: الرفع اليدوي يحتاج تكراراً عند كل تعديل،
        أما الرفع عبر Git فيتحدث الموقع تلقائياً عند كل push.
        (أجب "أظهر الجواب" إن أردت التأكد)
